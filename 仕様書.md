## 1. 概要

こわ面白い系コンテンツ（SNS投稿・短編小説・記事）を自動生成・投稿する統合アプリ。  
Claude API（Anthropic）を核とした3段階パイプライン（リサーチ → 執筆 → 整合性チェック）で、AIっぽさを排除したリアルな文章を生成する。

### 主な特徴
- Threads自動投稿、X用クリップボードコピー
- X記事はGoogleの検索結果に表示されるため、全コンテンツでSEOを意識した生成を行う
- 品質スコア（100点満点）で投稿品質を自動評価
- 伸びたパターンを学習して次回の生成に自動反映

## 7. 文章生成パイプライン（agents.py）

全コンテンツ（投稿・小説・記事）共通の3段階パイプライン：

```
① リサーチ（_research_idea）
    ↓ ネタ・ジャンルをもとに、登場人物・場所・時系列・外せない要素を箇条書きで整理
② 執筆
    ↓ リサーチメモ + ルール群を踏まえて本文を生成
③ 整合性チェック（_consistency_check）
       リサーチメモと本文を照合し、名前・時系列・設定の矛盾があれば修正して返す
```

### プロンプトに注入するルール群

| ルール | 概要 |
|---|---|
| `ANTI_AI_RULES` | AIっぽい表現を禁止。断定口調・実話風・ゾワッとするオチを必須化 |
| `QUALITY_BOOST_RULES` | 品質スコアで高得点を取りやすい文章作法（1文15〜50字・不気味な語彙・AI口調禁止） |
| `SEO_RULES`（記事） | Google検索を意識。タイトルタグ・スニペット最適化・見出しキーワード配置 |
| `NOVEL_SEO_RULES`（小説） | 同上・小説向け調整版。タイトル・書き出しを検索スニペット前提で書かせる |
| `HORROR_LEVEL_INSTRUCTIONS` | 怖さレベル1〜5に応じた強度指示 |
| `X_POLICY_RULES` | Xポリシー配慮モード時に注入。過激表現抑制・免責文付与 |
| `IMI_KOWAI_RULES` | 「意味がわかると怖い」ジャンル専用。答えを書かずに隠す構造 |
| `OMO_KOWAI_RULES` | 「おも怖い」ジャンル専用。コメディ→ゾワッの一発転換構造 |

### SEOについての背景
X（旧Twitter）の記事がGoogle検索結果に表示されるようになったため、記事・小説のタイトル・書き出しはGoogle検索からの流入を意識した設計にしている。

## 8. 品質スコア（100点満点）

`app.py > calc_quality_score()` で計算。

| 評価項目 | 加点/減点 |
|---|---|
| ベーススコア | 40点 |
| 適切な文字数（100〜500字） | +20点 |
| 1文の平均長（15〜50字） | +10点 |
| 不気味な語彙の密度（死・影・声など） | 最大+15点 |
| textstat 読みやすさ | 最大+5点 |
| AIっぽい表現（「なんと」「実は」「まとめると」等） | -5点/個 |

| スコア | 色 | 意味 |
|---|---|---|
| 75点以上 | 🟢緑 | 良好 |
| 50〜74点 | 🟡黄 | 普通 |
| 50点未満 | 🔴赤 | 要改善 |

## 11. 文体学習機能（aozora.py）

青空文庫（著作権切れ日本文学）または現代作家の文体をClaude APIで分析し、生成時に適用できる。

**対応作家（青空文庫）**: 江戸川乱歩・夢野久作・芥川龍之介・小泉八雲・泉鏡花・太宰治・谷崎潤一郎・坂口安吾・中島敦・幸田露伴

**対応作家（現代・Claude知識ベース）**: 20名超（伊坂幸太郎・東野圭吾・湊かなえ・綾辻行人・貴志祐介など）

文体プロファイルはDBの `scheduler_config` テーブルに `style_profile_{作家名}` キーで保存される。

## 12. 自動投稿スケジューラ（scheduler.py）

`APScheduler` を使用。アプリ起動時にバックグラウンドで動作。

- **動作条件**: サイドバーの「自動投稿モード」がON
- **動作内容**: 投稿予定時刻を過ぎたキューのアイテムを Threads APIで自動投稿
- **品質フィルター**: 品質スコアが設定閾値（デフォルト70点）を下回るアイテムはスキップ

## 13. トレンドリサーチ（agents.py）

RSSフィード（NHK・朝日・読売・毎日・産経・Yahoo!ニュース・ライブドア等）からリアルタイムでニュースを取得し、ホラー系ネタへ変換する。Serper APIキーがある場合はWeb検索も使用。

対応カテゴリ: ニュース系・Yahoo!ニュース・ライブドア・テック/IT系・怖い話系（専門サイト）

## 14. 認証（auth.py）

`APP_PASSWORD` 環境変数が設定されている場合のみ有効。  
パスワードが一致するとセッション中ログイン状態が維持される。未設定の場合は認証なしで全機能にアクセスできる。

---

## 15. AIっぽさ排除のルール（ANTI_AI_RULES）

生成コンテンツ全体に適用される絶対ルール：

- 断定口調（〜だ、〜した、〜だった）のみ使う
- 「実は」「なんと」「驚くことに」「まとめると」「つまり」「でしょう」「かもしれません」は一切使わない
- 綺麗にまとめない。オチは説明せずに感じさせる
- 実話風の語り口を徹底する
- 起承転結＋最後にゾワッとするオチを必ず入れる
